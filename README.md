# Алгоритм динамического ценообразования для ритейла
## О проекте
Это учебный проект, в котором создается сервис для автоматического расчета цен на товары. Сервис анализирует историю продаж и цены конкурентов, а затем предлагает новую цену, которая должна увеличить выручку.
## Состав проекта
Проект состоит из 4 основных частей:
1. Генератор данных - создает синтетические данные о продажах
2. Модель оптимальной цены - рассчитывает математически оптимальную цену
3. Правила ценообразования - простые логические правила из задания
4. Интерфейс - веб-страница для работы с сервисом
5. Симуляция - показывает, как алгоритм будет работать день за днем
## Как установить и запустить
### Требования
- Python 3.8 или новее
- Библиотеки: pandas, numpy, matplotlib, streamlit, plotly
### Пошаговая установка
1. Скачайте проект или склонируйте репозиторий:
	git clone [ссылка на репозиторий]  
	cd pricing_algorithm
2. Установите необходимые библиотеки:
	pip install pandas numpy matplotlib streamlit plotly
3.  Запустите приложение:
	streamlit run sprint4_mvp_complete.py
4. Откроется браузер с интерфейсом программы

## Как пользоваться программой
### Боковая панель (слева)
- **Параметры данных** - можно изменить количество дней и товаров
- **Кнопка "Сгенерировать новые данные"** - создает новый набор данных
- **Выбор товара** - переключатель между разными товарами
- **Симуляция** - выбор количества дней для прогноза и кнопка запуска
### Основной экран
- **Общая статистика** - сколько всего записей, товаров, дней
- **Анализ товара** - графики зависимости продаж от цены и функция выручки
- **Оптимальная цена** - показывает, какая цена принесет максимум выручки
- **Результаты симуляции** - таблица и график работы алгоритма по дням
## Как работает алгоритм
### 1. Генератор данных
Создает данные по формуле из задания:
	$$Продажи = 100 - 2 \times Цена + Случайный\;шум$$
Это означает, что при повышении цены на 1 рубль продажи падают на 2 штуки. Случайный шум делает данные похожими на реальные.
### 2. Модель оптимальной цены
Находит формулу зависимости продаж от цены для каждого товара:
	$$Продажи = A - B \times Цена$$
Затем рассчитывает цену, при которой выручка максимальна:
	$$Оптимальная\;цена = \frac{A}{2 \times B}$$
### 3. Правила ценообразования
Два простых правила из задания:
- **Правило 1**: Если цена конкурента ниже нашей на 10% или больше, снижаем нашу цену на 5%
- **Правило 2**: Если вчера не было продаж, снижаем цену на 1 рубль
### 4. Симуляция времени
Программа берет последний день из истории и начинает "проигрывать" будущие дни:
- День 1: применяет правила к текущей цене → получает новую цену
- Прогнозирует продажи при новой цене
- День 2: берет новую цену и снова применяет правила
- И так далее на выбранное количество дней

## Возможные ошибки и их решение

- **Ошибка: "streamlit не найден"**  
  Решение: Установите streamlit командой `pip install streamlit`

- **Ошибка: "plotly не найден"**  
  Решение: Установите plotly командой `pip install plotly`

- **Ошибка: "Файл не найден"**  
  Решение: Запускайте программу из той папки, где лежит файл, или укажите полный путь

- **Приложение не открывается в браузере**  
  Решение: После запуска в терминале появится ссылка вида `http://localhost:8501` — скопируйте ее в браузер вручную
## Критерии проверки (для преподавателя)
1. **Генерация данных** - создается CSV файл со 100 днями и 5 товарами
2. **Эластичность** - на графике видно, что при росте цены продажи падают
3. **Правила** - алгоритм меняет цены по двум заданным правилам
4. **Оптимальная цена** - рассчитывается математически правильная цена
5. **Симуляция** - видно, как цена меняется день за днем
6. **Интерфейс** - все работает через веб-страницу, не требуется командная строка
